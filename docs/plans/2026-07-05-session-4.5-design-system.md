# Session 4.5 — Design System (Evergreen) + bộ component dùng chung

## Context (vì sao làm việc này)

Người dùng thiết kế giao diện mới trong **Claude design** (phong cách "Evergreen": xanh lá,
card bo tròn, nút pill, font Nunito) và đã export ra `design/Sugar App.dc.html`. Nhưng app thật
vẫn dùng theme **xanh dương** cũ (`src/ui/theme/colors.ts` → `primary: #208AEF`), nên build lên
thấy "y như cũ". Design chỉ là bản HTML tham chiếu, **không tự áp vào code**.

Ngoài ra, mỗi màn hình hiện **tự viết lại StyleSheet** cho card/chip/button/segmented-control
(xem `log-reading-form.tsx` ~800 dòng, `reading-list-item.tsx`, các screen) — lặp code, khó
giữ đồng bộ khi đổi design.

**Mục tiêu session này (làm TRƯỚC khi hoàn tất Session 5 Trends):**
1. Dịch design token từ file HTML sang `src/ui/theme` (một nguồn sự thật cho app).
2. Nạp font Nunito.
3. Dựng **bộ component "ngu" (presentational) dùng chung** để các màn hình bự tái sử dụng.
4. Retrofit các màn hình đã build (Log, History, Detail, Edit) sang theme + component mới.

Kết quả: sau session này toàn bộ app hiện có đồng bộ với design; Session 5 (Trends) và các
session sau (Settings, Export, Onboarding) build thẳng trên bộ component này, code sạch.

## Thay đổi cụ thể

### 1. Dependencies
- Thêm `@expo-google-fonts/nunito` (miễn phí, OFL). `expo-font` đã có sẵn trong Expo.

### 2. Theme (`src/ui/theme/`)
- `colors.ts`: viết lại `palette` sang bảng Evergreen ở trên. **Giữ nguyên TÊN các semantic
  token** (`primary`, `surface`, `card`, `text`, `textMuted`, `border`, `inRange`, `high`, `low`…)
  để component cũ không vỡ — chỉ đổi giá trị. Thêm token mới:
  `primaryButton: #1B2B24`, `onPrimaryButton: #fff`, `surfaceMuted: #E9F5EF`,
  `accentBlue/Purple/Amber/Orange`, `warnBg: #FDECE4`, `outOfRange: #E8622C`, và
  `mealColor` map (Breakfast/Lunch/Dinner/Snack).
- `typography.ts`: thêm `fontFamily` tokens theo cân nặng — **quan trọng**: với custom font trên
  RN, `fontWeight` không đổi được nét đậm, phải chọn đúng file font. Thêm:
  ```
  fontFamily = {
    regular: 'Nunito_400Regular', semibold: 'Nunito_600SemiBold',
    bold: 'Nunito_700Bold', extrabold: 'Nunito_800ExtraBold', black: 'Nunito_900Black',
  }
  ```
  Component mới dùng `fontFamily` thay vì `fontWeight`.
- `spacing.ts`: thêm `radius.card = 20`.

### 3. Nạp font + default font (`app/_layout.tsx`)
- Dùng `useFonts` từ `@expo-google-fonts/nunito` (Regular/SemiBold/Bold/ExtraBold/Black).
- Gộp `fontsLoaded` vào cổng gate hiện có (cạnh `isDbReady`/`isInitialized`) → `<BootSpinner/>`
  cho tới khi font xong.
- Đặt default font cho `<Text>`/`<TextInput>` chưa migrate (ví dụ set `Text.defaultProps.style`
  = fontFamily regular) để chữ nào chưa đổi cũng thành Nunito, tránh "nửa cũ nửa mới".

### 4. Bộ component dùng chung — thư mục mới `src/ui/components/ui/` (+ barrel `index.ts`)
Component "ngu", chỉ nhận props + gọi callback, không chứa business logic:
- `AppText` — Text bọc sẵn Nunito, prop `variant` (title/heading/body/label/caption) + `weight`.
- `Button` — full-width pill; variant `primary` (nền `#1B2B24`), `accent` (xanh), `ghost` (chữ),
  `dangerOutline`; hỗ trợ icon + trạng thái loading. → thay `saveButton`, nút DONE/Delete/modal.
- `Card` — nền trắng, radius 20, shadow mềm. → thay `valueCard`, container row, card detail.
- `SectionLabel` — nhãn in hoa chữ mờ (lặp khắp nơi).
- `Chip` — pill chọn/lọc (active/inactive, màu tuỳ biến). → thay mealChip, filter chips.
- `SegmentedControl` — hộp pill 2+ đoạn. → thay unit toggle + Before/After timing.
- `Stepper` — nút −/＋ với giá trị giữa. → thay hours-after stepper (và target-range sau này).
- `ScreenHeader` — tiêu đề 24px/900 + subtitle + slot phải. → mọi tab.
- `IconTile` — ô vuông bo tròn màu chứa icon. → avatar row lịch sử, (settings sau này).
- `Badge` — pill nhỏ ("OUT OF RANGE", trạng thái).

Ghi chú: `StatCard` (thẻ số ở Trends) và `ListRow` (dòng Settings) sẽ được thêm bởi chính
session tiêu thụ chúng (Session 5 / Session 6) để tránh dựng component chưa dùng.

### 5. Retrofit màn hình đã build
- `app/(tabs)/index.tsx` (Log) → `ScreenHeader`.
- `src/ui/components/log-reading-form.tsx` (lớn nhất): valueCard→`Card`, unit toggle→
  `SegmentedControl`, meal chips→`Chip`, Before/After→`SegmentedControl`, stepper→`Stepper`,
  section label→`SectionLabel`, nút Save→`Button`. Giữ nguyên toàn bộ logic (validate/save/picker).
- `app/(tabs)/history/index.tsx` → `ScreenHeader` + `Chip` cho filter.
- `src/ui/components/reading-list-item.tsx` → `Card` + `IconTile` (màu theo meal) + `Badge`.
- `app/(tabs)/history/[id]/index.tsx` (detail) → `Card` + `Button`.
- `app/(tabs)/history/[id]/edit.tsx` → dùng lại `LogReadingForm` nên hưởng lợi tự động.
- `app/(tabs)/_layout.tsx` → token màu tự đổi sang xanh; set `tabBarLabelStyle.fontFamily`.

### 8. Cập nhật `CLAUDE.md` (quy ước để các session sau tuân theo)
Thêm mục **"Design System"** vào phần Conventions:
- Nguồn tham chiếu hình ảnh: `design/Sugar App.dc.html` (phong cách Evergreen). Sửa design ở
  Claude design → export đè file này → nhờ Claude diff & cập nhật `src/ui/theme` + component.
  Không sửa token bằng cách đoán; luôn đối chiếu file design.
- **Bắt buộc dùng lại** primitive trong `src/ui/components/ui/` (Button, Card, Chip,
  SegmentedControl, SectionLabel, Stepper, ScreenHeader, IconTile, Badge, AppText). **Không**
  viết StyleSheet inline cho card/nút/chip nữa; cần biến thể mới thì mở rộng primitive.
- Màu/khoảng cách/bo góc/font: chỉ lấy từ `src/ui/theme` (đừng hardcode hex trong component).
- Chữ đậm dùng `fontFamily` (Nunito_*), KHÔNG dùng `fontWeight` (custom font trên RN bỏ qua nó).
- Icon: dùng Ionicons (`@expo/vector-icons`).
